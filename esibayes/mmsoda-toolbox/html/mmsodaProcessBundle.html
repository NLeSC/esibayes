mmsodaProcessBundle
Syntax

bundle = mmsodaProcessBundle(conf,constants,bundle)

Description

bundle = mmsodaProcessBundle(conf,constants,bundle)

This function receives a bundle of tasks, which can either be a bundle of 'model' tasks, or a bundle of 'objective' tasks. mmsodaProcessBundle then extracts the relevant information from the fields of the bundle-of-tasks structure array, and constructs the input arguments to either the dynamic model, or to the objective function. Once the input arguments have been constructed, mmsodaProcessBundle calls either the model or one or more objective functions, then waits for the output, and fills in the relevant information in the bundle-of-tasks structure array. When all tasks have been evaluated, mmsodaProcessBundle passes the bundle-of-tasks (or rather, a bundle-of-results) back to the 'Master' process (when running in parallel) or back to the caller function (when running sequentially).

Note that when running sequentially, mmsodaProcessBundle is part of the 'Master' process (i.e. it is a subfunction of mmsoda), but when running in parallel, mmsodaProcessBundle is a process in its own right (albeit a 'Worker' process).
