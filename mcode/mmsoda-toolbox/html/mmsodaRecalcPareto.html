mmsodaRecalcPareto
Syntax

[sequences,complexes,propChildren] = mmsodaRecalcPareto(conf,sequences,complexes,propChildren)

Description

[sequences,complexes,propChildren] = mmsodaRecalcPareto(conf,sequences,complexes,propChildren)

Pareto scores are fundamentally different from 'normal' objective scores, because their value changes as more points join the population. This means that the Pareto scores (but not the objective scores) need to be re-calculated every so often. This is why mmsodaRecalcPareto differs from mmsodaCalcPareto; although they are fundamentally the same, the latter does not take into account any existing Pareto scores, whereas the former does.

Note that mmsodaRecalcPareto takes a small liberty with calculating the Pareto scores: only the last rows of sequences are included in recalculating the Pareto scores, since these are the relevant values in accepting or rejecting samples from propChildren in the Metropolis scheme. While this greatly increases the efficiency of the optimization, it can lead to confusing Pareto scores in figures as well as in the output arguments. Note that this is easily mitigated by calling mmsodaRecalcPareto once more before any visualization or analysis (but then using all rows of the sequences array) instead of just sequences(end,:,:).
